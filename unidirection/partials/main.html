Unidirectional Association Example App

An example of a frontend web app with constraint validation and unidirectional association management, built with AngularJS

This example app is accompanied by a tutorial. See also the project page.

    For playing with the app, you may first want to and then view it (e.g. by choosing "Manage books" and then "List all books").

    This example app for Part 3 of our 5-part tutorial takes care of the object types Author, Publisher and Book as well as the bidirectional associations between Book and Author and between Book and Publisher. The app supports the four standard data management operations (Create/Read/Update/Delete). It extends the example app of Part 2 by adding code for handling unidirectional associations, but it needs to be enhanced by adding further important parts of the app's overall functionality:

    • Part 4: Handling bidirectional associations between object types.
    • Part 5: Handling subtype (inheritance) relationships between object types.

    For reading more about the other example apps of this 5-part tutorial, see our project page.
